# Telegram-бот для парсинга Notion

Бот принимает ссылку на публичную страницу Notion, раскрывает все блоки и комментарии, собирает текст и ссылки и отправляет их в Telegram. При повторных запросах по той же ссылке бот показывает только новые и изменённые темы.

## Что делает
- Команды: `/start`, `/parse <url>`, `/raw <url>`, `/test`, `/clear`.
- Автообработка сообщений с `notion.so`/`notion.site` без команды.
- По умолчанию парсит страницу через Notion `recordMap`: напрямую читает блоки, rich-text, discussion-id и комментарии без запуска браузера. Это быстрее и точнее восстанавливает места ссылок/слэш-команд.
- Puppeteer оставлен как fallback: если `recordMap` не загрузился или нужен debug старого DOM-пути, скрейпер может снова прокручивать страницу, раскрывать toggles, открывать попапы комментариев и собирать ссылки/заметки.
- Разбивает контент на темы (`### заголовки`), считает хэши и кеширует, чтобы при повторе показывать только добавленные/изменённые блоки.
- Отправляет ссылки отдельно от текста, удаляет дубликаты, ставит эмодзи по типу ссылки (видео/аудио/фото/другое).
- В `/raw` отдаёт исходный текстовый дамп (Markdown) всей страницы.
- В `/parse` сохраняет файл с заголовками (чистые строки без хвостов вида `(1)`) и кладёт его в локальный HTTP-эндпоинт `GET /headlines?url=<notion_url>` для Figma-плагина (в чат больше не отправляет автоматически).

## Как устроено
- `notion-scraper.js` — основной вход. Сначала пробует быстрый `recordMap`-скрейп через `notion-recordmap-scraper.js`; если Notion API недоступен, падает обратно на Puppeteer.
- `notion-recordmap-scraper.js` — ходит в Notion `loadPageChunk`, догружает toggle-секции, читает структурные `block/discussion/comment` записи и формирует Markdown всей страницы. Он сохраняет реальные границы выделений, поэтому URL и `/комментарии` вставляются ближе к ручному эталону.
- Важный нюанс: нужная информация не всегда лежит в HTML/DOM сразу. Notion лениво подгружает свернутые блоки и комментарии, но эти данные доступны в `recordMap` через те же внутренние запросы, которыми пользуется сама страница. Новый скрейпер догружает секции запросами к `loadPageChunk`, вместо того чтобы имитировать клики по toggle и комментариям в браузере.
- `NOTION_PUPPETEER_ONLY=1` — принудительно включает старый Puppeteer-путь для отладки.
- `notion-scraper.js` в fallback-режиме запускает Chromium через Puppeteer, подгружает `notion-script.js` и возвращает Markdown всей страницы.
- `notion-script.js` (выполняется в браузере):
  - скроллит страницу, раскрывает свёрнутые блоки;
  - для каждого блока ищет заголовок, текст, ссылки, явные маркеры-комментарии (`/...`) и содержимое попапов комментариев;
  - формирует Markdown вида `# <title>`, далее `### <topic> (N)` со списком ссылок/комментариев и телом блока.
- `bot.js`:
  - слушает команды, валидирует Notion URL;
  - вызывает скрейпер и парсер, сравнивает полученные темы с кешом по md5-хэшу и рассылает: новые (`🆕`), изменённые (`✏️`), без изменений;
  - аккуратно отправляет сообщения с учётом rate limit (`safeSend` с retry) и сокращает слишком длинные тексты.

## Запуск
1. Требования: Node.js 18+ (Puppeteer подтянет собственный Chromium).
2. Установить зависимости:
   ```bash
   npm install
   ```
3. Создать `.env` в корне:
   ```env
   BOT_TOKEN=<токен телеграм-бота>
   ```
4. Старт:
   ```bash
   npm start
   ```

## Использование
- Отправьте боту ссылку на публичную страницу Notion — бот ответит статистикой и всеми найденными темами.
- Повторите ту же ссылку позже — бот покажет только новые/изменённые темы (кеш очищается по `/clear`).
- `/raw <url>` — вернёт файл `notion-content.txt` с полным дампом.
- `/headlines <url>` (или `/headers <url>`) — пришлёт заголовки одним или несколькими сообщениями (без файла), если ранее делали `/parse` для этой ссылки.
- `/test` — быстрая проверка доступности и размера кеша.

## Ограничения и заметки
- Работает только с публично доступными Notion-страницами.
- Кеш хранится в памяти процесса (сбрасывается при рестарте).
- Файл заголовков доступен локально по адресу `http://localhost:3131/headlines?url=<ссылка>` (порт меняется переменной `HEADLINES_PORT`). Для `localhost` URL плагин сам добавит `http://` и перекодирует параметр `url=<notion_url>`.
- Долгие страницы могут собираться до 10 минут (таймауты настроены на это).

## Дорожная карта
- Отслеживание изменений по расписанию с уведомлениями без ручного `/parse`.
- Поддержка приватных страниц (передача токена/куки Notion) и гибкие правила доступа.
- Постоянный кеш/история (SQLite/Redis) с дедупликацией и TTL.
- Более точный дифф: подсветка изменений в тексте, сравнение ссылок и вложений.
- Вложенные артефакты: загрузка изображений/файлов из Notion и отправка в Telegram.
- Логи/метрики и панель оператора (веб/бот-команды) для мониторинга ошибок и статуса.
- Упаковка в Docker и базовые тесты/CI для стабильных деплоев.
- Скриншоты и превью: автоматический захват скринов всех ссылок и вложений, превью для монтажки.
- Проверка источников: blacklist/whitelist доменов, подсветка подозрительных/рекламных/404 ссылок.
- Автоматические синопсисы: summary каждой темы и one-liner для монтажного листа.
- Кластеризация: группировка похожих тем и слияние дублей, чтобы убрать шум.
- Расстановка приоритетов: скоринг по “важность/срочность/новизна” + теги из текста и комментариев.
- Рекомендации: предлагать доп. материалы (видео, интервью, посты) из доверенных источников.
- Экспорт: готовый пакет для монтажки — текст, ссылки, скриншоты, теги, таймкоды, единым архивом или Google Drive/Notion-страницей.
- Интеграции: вебхук в продакшн-чат, автосоздание тасок (Jira/Linear), отправка в монтажный бриф.
- Обработка мультимедиа: скачивание аудио/видео/изображений, спринты транскрибации/описания (ASR+OCR).
- Live-режим: “режиссёрский пульт” — стрим обновлений в чат и кнопки “в работу/отклонить/в архив”.
- Политики качества: проверки на длину, наличие фактуры, конфликтные данные, авторство, ссылки без контекста.
- Самообучение: память о правках редакции (кто что отклонял/принимал) и автоподстройка подсказок/фильтров.
- Нормализация ссылок: очистка utm/redirect, проверка HTTPS, выделение домена для политики доверия.
- Профили подписок: авторассылки по выбранным темам/тегам в разные чаты/команды.
- Версионирование тем: история правок/диффов с быстрым rollback к прошлой версии страницы.
- Очереди/воркеры (BullMQ/RabbitMQ): параллельная обработка, retry, backpressure.
- Публичный HTTP API и CLI для вызовов без Telegram (интеграции с пайплайнами).
- Шаблоны выдачи: “полный разбор”, “короткий дайджест”, “только ссылки”, “материалы в монтаж”.
- Мультибот/мультичат: роли (редактор/корр/гость), ACL на команды и каналы уведомлений.
- Режим “черновик выпуска”: сбор тем в единый бриф, сортировка и экспорт в один пакет.
- Авторазметка: теги (NER/темы), язык, тональность, флажок “нужна проверка”.
- Аналитика источников: топ доменов, аптайм, share-of-voice, всплески подозрительных ссылок.
- Бэкапы/зеркала: хранение дампов и снимков страниц в S3/GCS с дедупликацией.
- Observability: OpenTelemetry/Prometheus, трассы, алерты в чат при ошибках/просадках скорости.
- Переключаемые движки браузера (Puppeteer/Playwright + fallback) и адаптивные тайминги.
- Плагины обработчиков ссылок (YouTube/VK/Twitter/Instagram): richer-превью и метаданные.
- Тестовый контур: фикстуры страниц, e2e через headless, контрактные тесты парсера.
